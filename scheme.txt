## テーブル
- 車両・速度・出力管理テーブル
car_addr int
speed int (これの最大値を最高速度とする)
level int (上に同じ)

- コントローラ・アドレス(車両)管理テーブル (同じコントローラなら区別しない、早く認識した順)
car_addr int
controller_code text

- キーマップ(ボタンとその動作)

192.168.3.190に有線は固定した。ID:root pass:dietpi
SSID:MultiMascon-[ランダム] PSK:12345678

DHCPしっかり書く
root@DesktopStation:~# sudo vim /etc/dnsmasq.conf
root@DesktopStation:~# sudo /etc/init.d/dnsmasq restart

flask run --host=0.0.0.0 --port=80

出荷時は、
- / をroに変更
- /mnt/multimasconをroに(OTA時のみrwにする)
- hostapd.confを初期化
- マニュアル作る
- シール貼る

エラーログファイルをdatabase領域に置く
キーマップ（どのボタンがどの動作）sqlite

パーティションは
- ext4(DietPi) ro
- btrfs(multimascon) ro
- btrfs(database) rw
- FAT32(RESCUE)

うまくソフトやDBが読み込めなかったら、btrfsの復旧をする
OTAアップデートも可能、バックアップ用にも配置する
